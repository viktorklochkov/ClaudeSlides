Skills System

Custom Skills Deep Dive

skill definition — .claude/skills/review.yml 
name: code-review
description: Review PR with project conventions
trigger: /review
steps:
  - tool: Bash
    command: gh pr diff
  - tool: Read
    glob: "CLAUDE.md"
  - prompt: |
      Review this PR against our conventions
      in CLAUDE.md. Check for:
      - Security vulnerabilities
      - Performance regressions
      - Missing test coverage
      - Style guide violations
      Provide actionable feedback.

How Skills Work

  • Declarative — Define workflows in YAML, not code
  • Triggered by command — Invoke with /review in the CLI
  • Multi-step — Chain tools, reads, and prompts together
  • Project-scoped — Lives in .claude/skills/ alongside your code
  • Shareable — Commit to repo, every team member gets the skill
Pro tip: Combine with CLAUDE.md for context-aware automation that respects your team's conventions.
Skills are one of the most powerful differentiators. They let teams codify their workflows. Example: a /deploy skill that runs tests, builds, and deploys — or a /migrate skill that generates database migrations following your ORM conventions. Copilot has no equivalent to this.

MCP provides a standardized protocol for connecting AI agents to any external tool or data source — like USB for AI.

MCP configuration — .claude/mcp.json 
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y", "@modelcontextprotocol/server-postgres",
        "postgresql://localhost/mydb"
      ]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_..."
      }
    },
    "filesystem": {
      "command": "npx",
      "args": [
        "-y", "@modelcontextprotocol/server-filesystem",
        "/path/to/allowed/dir"
      ]
    }
  }
}
MCP (Model Context Protocol) is Anthropic's open standard for connecting AI tools to external systems. Think of it as a plugin system — but standardized across all MCP-compatible agents. Key point: this is NOT proprietary. Any tool can implement an MCP server, and any agent can connect to it. Currently 50+ community servers available for databases, APIs, file systems, and more.

Real-World Scenario

Complex Refactoring: A Side-by-Side

Task: Migrate authentication from session-based to JWT across a full-stack application

🤖 Claude Code

  • Explores entire auth codebase with sub-agents
  • Plans migration strategy in plan mode
  • Creates JWT middleware, updates routes, modifies frontend — in parallel
  • Writes migration tests before changing code
  • Runs test suite, fixes failures autonomously
  • Updates .env.example and docs
Result: Complete migration in ~15 minutes, fully tested

🖥 Copilot

  • Suggests JWT code per-file as you edit
  • You manually identify all files to change
  • No cross-file awareness of the migration
  • Test suggestions are file-scoped
  • You run tests manually, fix manually
  • Documentation updates are your responsibility
Result: Hours of manual work with inline suggestions

Your First Skill

Getting Started

Step 1 — Install Claude Code

# Install globally via npm
npm install -g @anthropic-ai/claude-code

# Navigate to your project
cd my-project

# Start Claude Code
claude

Step 2 — Create Your First Skill

# Create the skills directory
mkdir -p .claude/skills

# Create a commit skill
cat > .claude/skills/commit.yml << 'EOF'
name: smart-commit
description: Commit with conventional message
trigger: /commit
steps:
  - tool: Bash
    command: git diff --staged
  - prompt: |
      Generate a conventional commit message
      for the staged changes. Use the format:
      type(scope): description
EOF

Step 3 — Use the Skill

# Stage your changes
git add .

# Invoke the skill in Claude Code
> /commit

# Claude reads the diff, generates:
# feat(auth): add JWT token refresh logic
#
# - Add refresh token rotation
# - Update middleware to check expiry
# - Add tests for token lifecycle

Step 4 — Add CLAUDE.md

# CLAUDE.md

## Project: My App
- Stack: React + Node.js + PostgreSQL
- Tests: Jest + React Testing Library
- Style: Airbnb ESLint, Prettier
- Commits: Conventional Commits format
- Always write tests before implementation
- Use TypeScript strict mode
